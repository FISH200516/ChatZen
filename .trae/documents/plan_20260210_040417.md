# Implement Baidu AI Search API

Based on the investigation of the local `qianfan.pdf` (via web search of its contents since it's binary) and online documentation, I will implement the Baidu AI Search API integration.

## 1. API Definition (`WebSearchApi.kt`)
- **Endpoint**: `POST /v2/ai_search/web_search`
- **Base URL**: `https://qianfan.baidubce.com/`
- **Headers**: `X-Appbuilder-Authorization: Bearer <API Key>`
- **Request Structure**:
  ```json
  {
    "messages": [{"role": "user", "content": "query"}],
    "search_source": "baidu_search_v2",
    "resource_type_filter": [{"type": "web", "top_k": 10}]
  }
  ```
- **Response Structure**:
  ```json
  {
    "references": [
      {
        "title": "Page Title",
        "url": "Page URL",
        "content": "Snippet/Content"
      }
    ]
  }
  ```

## 2. Retrofit Client (`RetrofitClient.kt`)
- Update `baiduApi` to use the correct base URL: `https://qianfan.baidubce.com/`.

## 3. Repository Implementation (`WebSearchRepository.kt`)
- Implement `searchBaidu` function.
- Construct the request body with the fixed `search_source` and user query.
- Map the response `references` to the app's `WebSearchResult` model.

## 4. UI/Settings
- The existing `WebSearchSettingsScreen` already supports entering an API Key for Baidu. No UI changes are needed, just the backend logic connection.
