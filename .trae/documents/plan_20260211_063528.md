我将按照以下步骤实现 OpenClaw Web UI 集成功能：

### 1. 数据层与状态管理
- **SettingsRepository**:
  - 新增 DataStore Key `openclaw_web_ui_url`。
  - 实现 URL 的读取 (`getOpenClawWebUiUrl`)、保存 (`saveOpenClawWebUiUrl`) 和清除 (`clearOpenClawWebUiUrl`) 方法。
- **SettingsViewModel**:
  - 在 `SettingsUiState` 中添加 `openClawWebUiUrl` 字段。
  - 暴露保存和清除 URL 的方法供 UI 调用。

### 2. 新增 UI 页面
- **OpenClaw 配置页 (`OpenClawSettingsScreen.kt`)**:
  - 创建新页面，包含 URL 输入框、保存按钮和清空按钮。
  - 实现输入验证与保存逻辑。
- **WebView 全屏页 (`WebViewScreen.kt`)**:
  - 创建基于 `AndroidView` 的 WebView 页面。
  - 配置 WebView 以适配移动端（启用 JS、设置移动端 UserAgent、适配视口）。
  - 实现全屏沉浸式显示逻辑。

### 3. 现有页面修改
- **设置页 (`SettingsScreen.kt`)**:
  - 添加“使用 OpenClaw Web UI”入口按钮，点击跳转至配置页。
- **对话页 (`ChatScreen.kt`)**:
  - 监听 `SettingsViewModel` 中的 `openClawWebUiUrl` 状态。
  - 修改发送按钮交互，使用 `combinedClickable` 添加长按手势。
  - 长按逻辑：若已配置 URL，则导航至 WebView 页；否则不响应。

### 4. 导航与路由 (`ChatZenNavHost.kt`)
- 定义新路由：`openclaw_settings` 和 `webview_screen?url={url}`。
- 在 `NavHost` 中注册新页面。
- 配置 `WebViewScreen` 为全屏模式（隐藏 BottomNavigation 和 TopAppBar）。

### 5. 验证计划
- 验证设置页 URL 的保存与读取是否持久化。
- 验证对话页发送按钮的长按是否能正确跳转。
- 验证 WebView 是否能正确加载网页并适配手机显示。
