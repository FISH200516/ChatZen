## 2. 修复服务提供商界面返回手势动画
解决服务商设置页面无法触发系统预测返回动画（Predictive Back Gesture）的问题。

### 修改文件：`ui/screens/ServiceProvidersScreen.kt`
- **优化 BackHandler**：
    - 当前代码中 `BackHandler(enabled = true)` 始终处于启用状态，导致系统无法接管返回手势来展示退出动画。
    - **修改方案**：动态设置 `enabled` 状态。仅当处于子页面（如“自定义提供商详情”或“模型列表”）时启用拦截；当处于顶层列表时，禁用拦截，交由系统处理返回动画。

```kotlin
// 修改前
BackHandler(enabled = true) { ... }

// 修改后
val canGoBackInternal = uiState.selectedCustomProvider != null || uiState.selectedProvider != null
BackHandler(enabled = canGoBackInternal) {
    // 仅处理内部状态回退逻辑
    ...
}
```
