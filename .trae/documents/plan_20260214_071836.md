导致闪退的原因是 `R.mipmap.ic_launcher` 在 Android 8.0+ 设备上通常解析为 **自适应图标 (Adaptive Icon)**，其底层实现是 XML 格式。Jetpack Compose 的 `painterResource` 在某些情况下无法直接加载这种复杂的 XML 组合图标，从而导致应用崩溃。

**修复方案**

1.  **创建独立图标资源**：
    我们将从 mipmap 目录中提取一个高分辨率的静态图标图片（WebP 格式），将其复制到 `drawable` 目录并重命名为 `app_logo.webp`。这样可以确保加载的是一个纯图片文件，彻底避免 XML 解析问题。

2.  **更新代码引用**：
    修改 `ChatScreen.kt`，将引用的资源 ID 从 `R.mipmap.ic_launcher` 改为 `R.drawable.app_logo`。

**具体步骤**

1.  执行命令行：将 `app/src/main/res/mipmap-xxxhdpi/ic_launcher.webp` 复制到 `app/src/main/res/drawable/app_logo.webp`。
2.  修改 `ChatScreen.kt`：更新 `Image` 组件的 `painter` 参数。

这不仅能修复闪退，还能确保在所有 Android 版本上图标显示一致。
